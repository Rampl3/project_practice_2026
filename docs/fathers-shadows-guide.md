# Техническое руководство по реализации проекта Father’s Shadows

## Введение

Данное руководство описывает основные технологии и этапы реализации проекта Father’s Shadows на игровом движке Godot 4.

Проект представляет собой 3D игру с дорожным трафиком, диалоговой системой и процедурной генерацией уровня.

---

# Используемые технологии

| Технология | Назначение |
|---|---|
| Godot 4 | Игровой движок |
| GDScript | Основной язык программирования |
| Git | Контроль версий |
| Blender | Создание и редактирование 3D-моделей |
| Markdown | Документация |

# Реализация управления автомобилем

## Основная логика

Для движения автомобиля использовалась физическая модель Godot 4.

### Основные функции:

- ускорение;
- торможение;
- поворот;
- ограничение скорости.

### Пример структуры скрипта

```gdscript
func _physics_process(delta):
    velocity += transform.basis.z * speed
    move_and_slide()
```

# Реализация дорожного трафика

## Traffic Generator

Для генерации автомобилей использовался отдельный скрипт `Traffic Generator`.

### Возможности системы

- случайное появление машин;
- распределение по полосам;
- контроль дистанции между автомобилями.

---

# Реализация бесконечной дороги

## Road Manager

Для создания бесконечной дороги использовалась система тайлов.

### Принцип работы

1. Игрок движется вперед.
2. Старые сегменты удаляются.
3. Новые сегменты дороги создаются автоматически.
4. К дороге прикрепляются модели окружения.

# Система камер

Было реализовано несколько положений камеры:

- стандартная камера;

Переключение происходило автоматически в зависимости от скорости автомобиля.
